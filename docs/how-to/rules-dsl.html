Rules DSL Reference

The Rules DSL lets you override generation strategies for specific tables and columns. Rules files are YAML or JSON (version "1") passed via --rules <path>.

File Formats

YAML

version: "1"
tables:
  - table: "public.users"
    columns:
      - column: "email"
        strategy: "Email"

JSON

{
  "version": "1",
  "tables": [
    {
      "table": "public.users",
      "columns": [
        { "column": "email", "strategy": "Email" }
      ]
    }
  ]
}

Top-Level Fields

FieldTypeRequiredDescription
versionstringYesMust be "1".
tablesarrayNoZero or more table rule entries.

Table Rule Fields

FieldTypeRequiredDescription
tablestringYesQualified table name, e.g. "public.users" or "main.users".
columnsarrayNoColumn rule entries.

Column Rule Fields

FieldTypeDescription
columnstringColumn name. Required.
strategystringDataKind name (e.g. "Email", "Integer").
fixedValueanyEmit this exact value every row. Overrides strategy.
nullRatenumber [0,1]Fraction of rows emitting null.
seedOffsetintegerAdded to the global seed for this column's generator.
distributionmap<string,number>Weighted discrete distribution.
jsonPathsarrayPer-path strategies for JSON/JSONB columns.

fixedValue

- column: "status"
  fixedValue: "active"

- column: "is_verified"
  fixedValue: true

- column: "score"
  fixedValue: 100

fixedValue takes the highest precedence — overrides strategy, distribution, and compliance profile masking.

nullRate

- column: "middle_name"
  nullRate: 0.7

70% of rows will be null. The remaining 30% use the inferred or specified strategy.

Combine with distribution:

- column: "tier"
  nullRate: 0.1
  distribution:
    silver: 0.5
    gold:   0.3
    platinum: 0.2

seedOffset

Decorrelates columns that would otherwise produce correlated output:

- column: "first_name"
  strategy: "FirstName"
  seedOffset: 1

- column: "last_name"
  strategy: "LastName"
  seedOffset: 2

distribution

- column: "country"
  distribution:
    US: 0.6
    GB: 0.2
    DE: 0.1
    FR: 0.1

Weights are normalised at generation time. If they sum to less than 1.0, the remainder uses a random generated value.

JSON Path Strategies

YAML

- column: "metadata"
  jsonPaths:
    - path: "$.email"
      strategy: "Email"
    - path: "$.address.city"
      strategy: "String"
    - path: "$.score"
      strategy: "Integer"
    - path: "$.tag"
      fixedValue: "synthetic"
    - path: "$.optional_field"
      strategy: "String"
      nullRate: 0.5

JSON

{
  "column": "metadata",
  "jsonPaths": [
    { "path": "$.email", "strategy": "Email" },
    { "path": "$.address.city", "strategy": "String" },
    { "path": "$.score", "strategy": "Integer" },
    { "path": "$.tag", "fixedValue": "synthetic" },
    { "path": "$.optional_field", "strategy": "String", "nullRate": 0.5 }
  ]
}

Paths use dollar-dot notation: $.field, $.parent.child. Array indexing is not supported.

JsonPath Rule Fields

FieldTypeDescription
pathstringDollar-dot path. Required.
strategystringDataKind name. Defaults to "String".
fixedValueanyFixed value for this path.
nullRatenumber [0,1]Probability path is omitted.

Precedence Merge

When multiple rule sources are combined, Concoction merges in priority order:

global defaults  <  project defaults  <  table rules  <  column rules

More specific rules win. Merging is performed by IRuleConfigurationService.Merge().

Validation

The CLI validates rules before generation. Errors are printed to stderr; exit code 1.

  • version must be "1"
  • table and column must be non-empty strings
  • nullRate must be between 0 and 1
  • strategy must be a recognised DataKind name

Complete YAML Example

version: "1"
tables:
  - table: "public.users"
    columns:
      - column: "email"
        strategy: "Email"
      - column: "name"
        strategy: "Name"
      - column: "status"
        fixedValue: "active"
      - column: "discount"
        nullRate: 0.3
        distribution:
          silver:   0.5
          gold:     0.3
          platinum: 0.2
      - column: "preferences"
        jsonPaths:
          - path: "$.email"
            strategy: "Email"
          - path: "$.address.city"
            strategy: "String"
          - path: "$.score"
            strategy: "Integer"
          - path: "$.tag"
            fixedValue: "synthetic"
          - path: "$.optional"
            strategy: "String"
            nullRate: 0.5

  - table: "public.orders"
    columns:
      - column: "status"
        distribution:
          pending:    0.4
          processing: 0.3
          shipped:    0.2
          cancelled:  0.1
      - column: "notes"
        nullRate: 0.6
        strategy: "Text"
