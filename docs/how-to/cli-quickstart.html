CLI Quick Start

Step-by-step guide to generating synthetic data for a SQLite database using all five CLI commands.

Prerequisites

  • .NET 10 SDK installed
  • Repository cloned and built: dotnet build Concoction.slnx
  • sqlite3 CLI (or any SQLite client)

Step 1: Create a Sample SQLite Database

sqlite3 sample.db <<'SQL'
CREATE TABLE departments (
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  name  TEXT NOT NULL
);
CREATE TABLE employees (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  department_id INTEGER NOT NULL REFERENCES departments(id),
  email         TEXT NOT NULL UNIQUE,
  name          TEXT NOT NULL,
  manager_id    INTEGER REFERENCES employees(id),
  status        TEXT NOT NULL DEFAULT 'active'
);
CREATE TABLE orders (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  employee_id INTEGER NOT NULL REFERENCES employees(id),
  amount      DECIMAL(10,2) NOT NULL,
  created_at  DATETIME NOT NULL
);
SQL

Step 2: Discover the Schema

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- discover \
  --provider sqlite \
  --connection "Data Source=./sample.db" \
  --database sample \
  --seed 42

Outputs JSON listing all tables with columns, PKs, FKs, and constraints.

Step 3: Profile the Schema

discover-profile surfaces diagnostics: self-referencing FKs, cycles, and unmapped types.

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- discover-profile \
  --provider sqlite \
  --connection "Data Source=./sample.db"

Exit code 1 if diagnostics found. Expect a diagnostic noting employees.manager_id is a self-referencing FK.

Step 4: Create a Rules File

Create rules.yaml in the project root:

version: "1"
tables:
  - table: "main.employees"
    columns:
      - column: "email"
        strategy: "Email"
      - column: "name"
        strategy: "Name"
      - column: "status"
        fixedValue: "active"
  - table: "main.orders"
    columns:
      - column: "amount"
        distribution:
          "9.99":  0.4
          "49.99": 0.35
          "99.99": 0.25

Step 5: Generate Data

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- generate \
  --provider sqlite \
  --connection "Data Source=./sample.db" \
  --database sample \
  --seed 42 \
  --rows 20 \
  --rules ./rules.yaml \
  --compliance-profile Default \
  --output ./artifacts

On success: Generated 60 rows across 3 tables.

Step 6: Validate Without Exporting

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- validate \
  --provider sqlite \
  --connection "Data Source=./sample.db" \
  --seed 42 \
  --rows 20

Exit code 0 = clean. Exit code 3 = validation issues (details on stderr).

Step 7: Export a Single Format

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- export \
  --provider sqlite \
  --connection "Data Source=./sample.db" \
  --seed 42 \
  --rows 20 \
  --format sql \
  --output ./sql-only

Supported --format values: json, csv, sql.

Sample SQL Output

INSERT INTO "main"."employees" ("id","department_id","email","name","manager_id","status")
VALUES (1,3,'alice@example.com','Alice Johnson',NULL,'active');
INSERT INTO "main"."employees" ("id","department_id","email","name","manager_id","status")
VALUES (2,1,'bob@example.com','Bob Smith',1,'active');
