1. Introduction & Concepts

Concoction is a deterministic synthetic data platform for .NET 10. Given a live database it discovers the schema, plans generation order, generates realistic synthetic rows, validates referential integrity, and exports results in CSV, JSON, and SQL.

TermMeaning
SeedInteger that makes generation deterministic. Same seed + same schema = identical output.
DataKindClassification of a column's data type (e.g. Email, Integer).
Rules DSLYAML or JSON configuration that overrides generation strategies per column.
Compliance ProfilePreset masking policy: Default, Healthcare, or Finance.
Generation PlanTopologically ordered list of tables, tracking cycles and self-references.
Validation IssueA detected constraint violation in generated data.
Run SummaryJSON artifact with table/row counts, timing, and validation issue count.

2. Schema Discovery

Concoction uses provider-specific adapters to introspect a live database. The discovery process is read-only.

Providers

Provider--provider valueNotes
SQLitesqliteReads sqlite_master and pragma_* views
PostgreSQLpostgresReads information_schema and pg_catalog

What is Discovered

For each table: name and schema, columns (name, SQL type, inferred DataKind, nullability, max length, precision/scale, default, allowed values), primary key, foreign keys, unique constraints, and indexes.

CLI

dotnet run --project ./Concoction.Cli -- discover \
  --provider sqlite \
  --connection "Data Source=./mydb.db" \
  --database mydb \
  --seed 42

Profile Command

discover-profile augments discovery with diagnostics: self-referencing tables, cycle edges, and unmapped column types. Exit code 1 if diagnostics found.

dotnet run --project ./Concoction.Cli -- discover-profile \
  --provider sqlite \
  --connection "Data Source=./mydb.db"

3. Generation Plan

Before generating data, Concoction builds a generation plan by analysing FK edges.

Topological Ordering

Tables are sorted so parent tables (referenced side of a FK) are generated before child tables (referencing side).

Cycle Detection

Cycles are recorded in GenerationPlan.Cycles. Concoction generates one side first (breaking the cycle at an optional FK), then backfills the cyclic reference.

Self-Referencing Tables

Tables with a FK pointing back to themselves are listed in GenerationPlan.SelfReferencingTables.

{
  "orderedTables": ["main.departments","main.employees"],
  "cycles": [],
  "diagnostics": [],
  "selfReferencingTables": ["main.employees"]
}

5. Rules DSL

Rules files let you override generation strategies for specific tables and columns. Supported in YAML or JSON, version "1".

Example

version: "1"
tables:
  - table: "public.users"
    columns:
      - column: "email"
        strategy: "Email"
      - column: "status"
        fixedValue: "active"
      - column: "tier"
        nullRate: 0.1
        distribution:
          bronze: 0.5
          silver: 0.3
          gold: 0.2

Column Rule Fields

FieldTypeDescription
columnstringColumn name. Required.
strategystringDataKind name to use.
fixedValueanyEmit this exact value every row.
nullRatenumber [0,1]Fraction of rows emitting null.
seedOffsetintegerAdded to the global seed for this column.
distributionmap<string,number>Weighted discrete value distribution.
jsonPathsarrayPer-path strategies for JSON/JSONB columns.

Precedence

global defaults < project defaults < table rules < column rules. More specific rules win. fixedValue always takes the highest precedence.

6. JSON Path Strategies

For JSON/JSONB columns, assign strategies to individual paths using dollar-dot notation.

- column: "preferences"
  jsonPaths:
    - path: "$.email"
      strategy: "Email"
    - path: "$.address.city"
      strategy: "String"
    - path: "$.score"
      strategy: "Integer"
    - path: "$.tag"
      fixedValue: "synthetic"
    - path: "$.optional"
      strategy: "String"
      nullRate: 0.5

Nested objects are constructed automatically from path segments. Array indexing is not supported.

JsonPath Rule Fields

FieldTypeDescription
pathstringDollar-dot path, e.g. $.email or $.address.city.
strategystringDataKind name. Defaults to "String".
fixedValueanyFixed value for this path.
nullRatenumber [0,1]Probability this path is omitted.

7. Compliance Profiles

ProfileValueDescription
DefaultDefaultNo masking.
HealthcareHealthcareHIPAA-style: masks Email, Phone, Name, Address, PostalCode.
FinanceFinancePCI-style: all Healthcare masks plus Currency/financial identifiers.
dotnet run --project ./Concoction.Cli -- generate \
  --provider postgres \
  --connection "Host=localhost;Database=db;Username=dev;Password=dev" \
  --compliance-profile Healthcare \
  --rows 1000

Precedence: fixedValue wins over compliance masking. Compliance profile wins over strategy for sensitive field categories.

8. Validation

After generation, Concoction validates constraints and records issues in GenerationResult.ValidationIssues.

CheckDescription
FK IntegrityEvery non-null FK value references an existing row in the parent table.
UniquenessUNIQUE columns have no duplicate values.
NullabilityNon-nullable columns have no null values.
Self-Ref RootRow 0 in a self-referencing table is handled correctly.
{ "table": "main.orders", "column": "customer_id",
  "reason": "FK value 999 not found in main.customers" }
CommandCodeMeaning
validate0No issues
validate3Validation issues found
generate2Validation issues found (data written)

9. Exports

The generate command always writes all three formats plus summary.json:

artifacts/
  csv/     main.users.csv, main.orders.csv, ...
  json/    main.users.json, main.orders.json, ...
  sql/     main.users.sql, main.orders.sql, ...
  summary.json

CSV

RFC-4180 compliant. Header row is column names. Nulls are empty fields.

JSON

Array of row objects per file.

SQL

INSERT statements. Uses NULL for nulls, TRUE/FALSE for booleans, proper SQL quoting.

summary.json

{
  "startedAt": "2024-06-01T10:00:00Z",
  "completedAt": "2024-06-01T10:00:01Z",
  "tableCount": 3,
  "rowCount": 150,
  "validationIssueCount": 0,
  "messages": []
}

10. Multi-Tenant Platform

Accounts

Top-level organisational unit. Members have roles: Member or Owner. Invitation flow: email + expiry token.

Workspaces

Scoped under Accounts. Contains connection catalog, secret references, agent instructions, and members with RBAC roles: Viewer / Editor / Admin.

Projects

Scoped under Workspaces. Holds database catalog entries and run history. Supports soft-delete (archive).

Governance

FeatureDescription
Account GroupsLogical groupings of members.
Allowed DomainsEmail domain allowlist for invitations.
Audit LogAppend-only log of significant events.

11. Agent Chat

Chat sessions let you interact with Concoction via natural language. Sessions are created per workspace.

Built-in Tools

ToolDescription
discover-schemaCalls ISchemaDiscoveryService, returns JSON summary of discovered tables and columns.
generate-dataRuns the full generation pipeline, returns JSON result with row and table counts.

Workspace-level agent instructions are prepended to every session in that workspace as a system prompt.

12. Workflows & Skills

Workflows define multi-step automation sequences triggered on demand via POST /workflows/{id}/run.

The custom skill registry supports OpenAPI contract ingestion — import an external API spec to expose its operations as callable workflow steps.

13. API Keys

All API keys use the cnc_ prefix. The plaintext secret is shown once at creation. The platform stores only the SHA-256 hash.

ScopeAccess
workspace:readRead-only access to workspace resources.
workspace:writeRead + write access.
adminFull administrative access.

Keys are created via POST /accounts/{accountId}/api-keys and revoked via DELETE /accounts/{accountId}/api-keys/{keyId}.

14. Self-Referencing FK Backfill

A self-referencing FK column points back to the primary key of the same table (e.g. employees.manager_id → employees.id).

Algorithm

  1. Row 0 (root): FK set to null (if nullable).
  2. Rows 1..N: Each row references row N-1 (chain structure).

If the self-referencing FK is non-nullable, row 0 cannot be null and a ValidationIssue is emitted. Use discover-profile to identify these tables early.
