Team-friendly deterministic run

dotnet run --project ./Concoction.Cli/Concoction.Cli.csproj -- generate \
  --provider sqlite \
  --connection "Data Source=./sample.db" \
  --database sample \
  --seed 42 \
  --rows 100 \
  --rules ./rules.yaml \
  --compliance-profile Default \
  --output ./artifacts

# Repeat with the same seed to reproduce the same dataset.

What Concoction is

A schema-aware synthetic data platform focused on realistic outputs, team workflows, and controlled reproducibility.

Schema Discovery

Read-only discovery captures tables, columns, keys, foreign keys, unique constraints, and indexes from SQLite or PostgreSQL.

Referential Integrity Engine

Topological planning and cycle diagnostics ensure parent/child generation order and safe handling of self-referencing tables.

Policy & Governance

Rules DSL, compliance profiles, and validation checks provide control over generated data quality and sensitivity behavior.

How teams use it

A practical flow from discovery to synthetic dataset delivery across local development, CI pipelines, and test environments.

1

Discover

Connect to a source database and inspect schema shape using discover or discover-profile.

2

Define Rules

Apply per-table and per-column generation overrides via YAML/JSON rules and optional compliance profiles.

3

Generate & Validate

Run deterministic generation with row targets and validate referential integrity, nullability, and uniqueness constraints.

4

Export & Integrate

Ship outputs as CSV/JSON/SQL and integrate via CLI, REST endpoints, or the TypeScript SDK for automation.
